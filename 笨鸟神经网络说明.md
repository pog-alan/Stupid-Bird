# 笨鸟神经网络说明

当前默认执行版本为 `笨鸟 v0.1`。
如果后续设计没有特别声明更高版本，均默认遵守 [笨鸟v0.1规范](d:/SB/文档/笨鸟v0.1规范.md)。

笨鸟神经网络（SB）是一种以存储驱动计算的神经网络架构，核心特点是稀疏信号路由。
它的基本思想是：存储空间不再只是被动查表，而是直接参与计算、匹配、传播与组合。

## 当前版本定位

笨鸟 v0.1 被限定为：

- 面向受限场景描述的对象—属性—关系—事件解释引擎
- 输入为短文本场景描述
- 输出为结构化场景解释，而不是自由文本生成
- 领域限定在局部场景异常与堆放理解

这意味着当前版本不以聊天、续写、开放域问答或通用推理为目标。

## 设计目标

笨鸟神经网络希望同时支持以下能力：

- 增量学习，而不是每次都整体重训
- 稀疏激活，而不是全局密集计算
- 推理时的多信号关联
- 让记忆、路由和匹配共同参与计算过程

## 核心修正

最初的设想是“每学到一个概念，就开辟一个新的神经元空间”。
这个方向是成立的，但未经约束会出现两个问题：

- 空间爆炸：细小变化也会不断生成新空间
- 算力爆炸：每个信号都可能向大量空间扩散

因此，修正后的版本加入了生命周期规则：

1. 只有当新颖度持续高于阈值时才创建新空间
2. 当相似度足够高时优先更新已有空间
3. 多个空间反复收敛后允许合并
4. 单个空间内部过于混杂时允许拆分
5. 长期贡献很弱的空间允许遗忘、归档或剪枝

## 系统结构

笨鸟神经网络可以分为五层：

1. 信号分解层
2. 稀疏路由层
3. 神经元空间图
4. 工作记忆与束搜索控制器
5. 输出评分层

### 1. 信号

信号是系统中的最小通信单元。
它同时携带语义信息和控制信息。

```text
信号 = {
  编号,
  类型,
  数值,
  标签,
  向量表示,
  能量,
  置信度,
  生存步数,
  来源路径
}
```

### 2. 神经元空间

神经元空间是局部的“计算-记忆”模块。
它既不是单个神经元，也不是一整层网络，而是围绕某个概念、关系、操作或情景形成的有界区域。

```text
空间 = {
  空间编号,
  空间类型,
  敏感标签,
  偏好信号类型,
  激活偏置,
  变换规则,
  连接关系,
  使用统计
}
```

推荐的空间类型包括：

- 概念空间：概念或特征抽象
- 关系空间：概念之间的关系
- 操作空间：可复用的操作或变换
- 情景空间：短期情景痕迹

### 3. 路由

一次输入会先被分解成多个主信号。
例如：

```text
输入 x -> {信号1, 信号2, 信号3, 信号4}
```

每个信号只激活最匹配的一小部分空间：

```text
激活分数 =
  0.55 * 信号与空间相似度 +
  0.20 * 上下文重叠度 +
  0.15 * 类型匹配度 +
  0.10 * 先验偏置 -
  路由代价
```

只有超过阈值且位于候选上限范围内的空间才会继续传播。
这是防止组合爆炸的第一道防线。

### 4. 局部变换

每个被激活的空间会根据局部规则或局部模块生成新的信号。
这些新信号不会立刻被接收，而是要先经过第二道筛选：

- 与当前工作记忆中的其他信号做相关性匹配
- 检查与当前上下文是否一致
- 检查剩余能量和生存步数是否足够

### 5. 全局选择

推理过程被视为在空间图上的稀疏搜索。
系统不会放任所有路径无限扩张，而是持续保留一组当前最优的局部假设。

```text
路径分数 =
  累积路由分数 +
  累积相关性分数 +
  输出置信度 -
  计算代价
```

最终分数最高的假设成为当前输出结果。

## 为什么存储可以参与计算

在笨鸟神经网络中，存储不是被动的，因为每个空间都会记录：

- 它对什么信号敏感
- 它如何变换匹配到的信号
- 它倾向于激活哪些后继空间
- 这些转移过去有多可靠

因此，“读取记忆 -> 触发路由 -> 发射信号 -> 关联匹配 -> 继续传播”本身就是完整的计算循环。
换句话说，保存下来的空间结构和连接关系，本身就构成了机器的一部分。

## 学习规则

### 创建 / 更新规则

对于每次训练片段：

1. 先将当前信号路由到已有空间
2. 如果最佳匹配空间足够强，则更新该空间
3. 如果新颖度依然很高，则分配新空间

建议的新颖度定义：

```text
新颖度 = 1.0 - 最大匹配分数(已有空间集合, 当前片段特征)
```

### 连接更新规则

如果空间 A 在成功路径中经常激活空间 B：

```text
连接(A, B) += 学习率 * 奖励 * 共激活程度
```

如果这种转移不再有效，就应当衰减它。

### 合并规则

当以下条件同时成立时，允许合并空间：

- 特征签名高度重叠
- 输出模式高度重叠
- 共激活频率高
- 在评估中冲突较低

### 拆分规则

当以下条件同时成立时，允许拆分空间：

- 激活频率长期较高
- 发射模式明显呈多峰分布
- 局部误差持续偏高

### 遗忘规则

当满足以下条件时，可以归档或剪枝空间：

- 使用率很低
- 正向贡献很弱或长期为负
- 附近存在持续压制它的更强空间

## 动态规划与缓存

只有在大量重复子问题存在时，动态规划才会特别有效。
笨鸟神经网络通常带有循环和分支，因此严格意义上的动态规划未必总能直接套用。
更合适的做法是“带缓存的稀疏搜索”。

缓存键可以定义为：

```text
(空间编号, 信号签名, 上下文签名)
```

缓存内容可以包括：

- 局部发射结果
- 局部分数贡献
- 在有限深度下的最优后续延展

因此，它更接近以下方法：

- 束搜索
- 有界图上的维特比式解码
- 记忆化图推理

而不是传统的表格式动态规划。

## 复杂度控制

笨鸟神经网络只有在分支受到严格控制时才有现实可行性。
推荐使用以下控制项：

- 每个信号的候选空间上限
- 每一轮保留的假设上限
- 每个空间的最大发射数
- 使用语义签名去重，阻断自强化回路
- 使用信号步数上限
- 设置硬激活阈值
- 设置硬相关性阈值
- 每次传播都衰减能量
- 周期性执行合并与压缩

近似推理复杂度可以写成：

```text
O(步数 * 束宽 * 活跃信号数 * 候选空间上限 * 最大发射数)
```

只有当这些因素都保持稀疏时，计算才是可控的。

## 仓库中的最小原型

当前仓库包含一个轻量级 Python 原型与一组文档：

- `sb/core.py`：信号、空间、路由、束搜索推理与增量学习接口
- `examples/minimal_demo.py`：展示多信号传播过程的小型规则示例
- `文档/笨鸟神经网络形式化规范.md`：关于生命周期、推理阶段和评估方式的形式化说明
- `文档/笨鸟v0.1规范.md`：当前版本的固定边界、任务定义、输入输出规范与成功标准
- `文档/笨鸟v0.1可实现设计.md`：当前版本的解析规则、评分公式、状态更新和实现模块划分
- `文档/笨鸟v0.1配置样例.json`：领域词表、场景权重、阈值和互斥关系样例
- `文档/笨鸟v0.1自动采集学习方案.md`：受限自动化采集与学习的流程与边界
- `文档/笨鸟v0.1采集配置样例.json`：自动采集白名单与阈值门控配置示例
- `sb/server.py`：常驻进程 + 简单 HTTP 对话接口
- `sb/proactive.py`：主动提问生成器
- `sb/dialog.py`：对话状态存储与轻量上下文
- `sb/vector_memory.py`：轻量向量记忆与近邻检索
- `sb/llm_bridge.py`：面向 LLM 的上下文构造层
- `sb/llm_runtime.py`：面向 LLM 的消息编排与响应约束层
- `文档/笨鸟LLM架构设计.md`：笨鸟作为 LLM 前置认知层的整体设计

这个原型刻意保持轻量。
它还不是完整的可训练神经网络，但已经把架构想法落实为可运行的接口和搜索流程。
当前版本还加入了转移先验、信号级去重和显式推理预算，使存储图谱能够影响计算，又不至于失控扩散。

## 向 LLM 方向的扩展

为了让笨鸟逐步走向更开放的语义理解与生成，当前版本新增了两层能力：

- 向量检索层：把概念、属性、关系、事件和场景组织成可检索记忆
- LLM 桥接层：把结构化结果和检索到的记忆打包成可直接送给生成模型的上下文

这意味着笨鸟不再只依赖词表精确命中，而是开始具备“近义召回 + 检索增强生成”的雏形。
当前实现仍然是轻量级的哈希向量索引，但接口已经为未来切换到真实嵌入模型预留好了位置。
在线学习升稳后的知识也会同步进入向量索引，因此检索层可以随着后台学习持续更新。
当前向量层已支持可替换嵌入后端，可从默认哈希编码平滑切换到本地句向量模型或外部嵌入服务。
当前本地句向量推荐起步配置为 `BAAI/bge-small-zh-v1.5`，并已支持检索指令前缀。

## 建议路线

1. 先在玩具任务上验证信号-空间抽象是否稳定
2. 再把基于规则的局部变换替换成可训练模块
3. 为大规模空间集合加入向量索引路由
4. 补齐自动合并、拆分和剪枝流程
5. 加入任务奖励与长程评估机制
6. 探索与事件驱动或类脑硬件的映射关系

## 总结

笨鸟神经网络最适合被定义为：

“一种以显式神经元空间为核心、以稀疏信号传播为推理方式、让概念、关系和操作共同存储并共同参与计算的存储驱动神经网络架构。”

这个版本保留了你最初的直觉，也让它更接近可以实现、可以实验、可以迭代的系统。
