# 笨鸟 v0.1 自动采集学习方案

本方案面向“有限自动化”，目标是让笨鸟在受限边界内自动扩充对象、属性、关系、事件与场景提示，而不发生语义漂移与空间爆炸。

## 原则

- 只允许白名单来源
- 自动化只产出候选，不直接进入稳定空间
- 任何稳定空间都必须可追溯、可回滚
- 出现冲突时自动降级为临时

## 处理流程

1. 白名单采集  
限定域名与起始页面，遵守 `robots.txt`，限制抓取深度与速率。

2. 去噪与清洗  
去除模板噪声、重复段落、过短文本；只保留正文内容。

3. 结构化抽取  
将文本映射为五类候选：对象、属性、关系、事件、场景提示。

4. 质量评分  
根据置信度与冲突惩罚将候选分为：  
`candidate` / `temporary` / `drop`

5. 受控入库  
只入 `临时` 或 `候选` 空间，稳定空间必须满足重复出现与多源一致性。

6. 稳定升级  
同一候选在多源出现且一致性高时，自动晋级为稳定。

7. 版本与回滚  
每次升级记录证据与来源，可回滚到上一个稳定版本。

## 阈值建议

- 进入 `candidate`：置信度 ≥ 0.85
- 进入 `temporary`：置信度 ≥ 0.70
- 晋级 `stable`：同一候选出现次数 ≥ 3 且来源 ≥ 2

## 安全边界

- 不采集社交媒体与论坛
- 不抓取登录后内容
- 不采集私有数据或个人敏感信息
- 只做“领域内扩展”，不拓展到新领域

## 与 v0.1 的接口

自动采集模块只输出候选结构，真实入库需经过质量门控。
稳定知识写入 `笨鸟v0.1配置样例.json` 的 `object_lexicon`、`attribute_lexicon`、`scene_hint_terms` 三类字段。

## 模块划分

- `sb/auto_crawl.py`：白名单采集
- `sb/extractor.py`：结构化抽取
- `sb/quality_gate.py`：阈值门控
- `sb/ingest.py`：入库与版本维护
- `sb/server.py`：常驻后台学习 + 简单 HTTP 接口
- `sb/proactive.py`：主动提问生成器
- `sb/dialog.py`：对话状态存储与轻量上下文

## 对话上下文融合

服务端会自动把同一 `session_id` 最近若干轮文本拼接为上下文，作为解析输入。
这样用户对主动提问的简短回答也能被还原到完整场景中。
