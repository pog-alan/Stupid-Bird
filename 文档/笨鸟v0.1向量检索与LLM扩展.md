# 笨鸟 v0.1 向量检索与 LLM 扩展

本扩展的目标不是把笨鸟立刻做成大语言模型，而是让它具备成为 LLM 前端语义记忆层的能力。

## 目标

- 让概念、属性、关系、事件、场景具备近义召回能力
- 让词表外输入也能找到相近空间
- 为后续生成模型提供结构化记忆上下文

## 当前实现

### 向量记忆

当前版本已升级为“可插拔嵌入后端”。
默认仍使用轻量级哈希 n-gram 向量编码，不依赖额外机器学习库，但现在也支持：

- `http`：对接外部嵌入服务
- `sentence_transformers`：对接本地句向量模型
- `hashed`：本地兜底

每条记忆记录都包含：

- `space_id`
- `space_type`
- `label`
- `text`
- `supports`

### 检索逻辑

对输入信号先做原有规则路由，再做向量近邻召回。
两路候选合并后进入统一打分流程。

### 嵌入后端

#### 1. hashed

优点：

- 零依赖
- 可直接运行
- 适合原型验证

缺点：

- 语义能力有限

#### 2. http

通过 HTTP POST 调用外部嵌入服务。
请求示例：

```json
{
  "texts": ["空地上堆着碎砖和木板"]
}
```

兼容两种常见响应格式：

```json
{
  "vectors": [[0.1, 0.2, 0.3]]
}
```

或：

```json
{
  "data": [
    {"embedding": [0.1, 0.2, 0.3]}
  ]
}
```

#### 3. sentence_transformers

如果本地已安装 `sentence-transformers`，可以直接加载句向量模型。
适合后续做更像 LLM 体系的检索层。

推荐中文起步配置：

- `BAAI/bge-small-zh-v1.5`
- 查询指令：`为这个句子生成表示以用于检索相关文章：`

如果你更重视效果，可以升到：

- `BAAI/bge-base-zh-v1.5`

当前实现已经支持：

- 查询和文档分开编码
- `query_instruction` / `passage_instruction`
- `device` / `cache_folder` / `batch_size`
- 缺包时自动回退到 `hashed`

### 动态同步

后台学习模块会将升为 `stable` 的知识写入知识库。
服务端刷新快照时，会同时读取：

- 在线配置中的稳定词表
- 知识库中的稳定条目

然后共同构建向量索引。
这意味着新学到的稳定对象、属性、关系、事件、场景提示，会自动进入向量检索层。

### LLM 桥接

新增一个 LLM 上下文构造层，输出：

- `system_prompt`
- `user_prompt`
- `structured_analysis`
- `retrieved_memories`

## 新接口

### 引擎接口

- `SBV01Engine.retrieve_memories(text, analysis)`
- `SBV01Engine.build_llm_payload(text, analysis)`

### HTTP 接口

- `POST /chat`：返回结构化分析、主动提问、检索记忆
- `POST /llm_context`：返回可直接交给 LLM 的上下文包

### 同步示例

- `python -m examples.v01_vector_sync_demo`
- `python -m examples.v01_embedding_backend_demo`
- `文档/笨鸟v0.1本地句向量配置样例.json`

## 后续升级方向

1. 用真实嵌入模型替换哈希向量编码器
2. 把在线采集到的稳定知识同步写入向量索引
3. 在生成层接入外部 LLM，形成 RAG 风格闭环
