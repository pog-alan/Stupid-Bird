# 笨鸟 v0.1 规范

## 1. 正式定位

### 名称

笨鸟（Stupid Bird）v0.1

### 版本定位

面向受限场景描述的对象—属性—关系—事件解释引擎。

### 核心目标

输入一段受限领域、短长度、结构相对清晰的场景描述文本，笨鸟输出：

- 对象集合
- 属性集合
- 关系集合
- 事件集合
- 候选场景解释
- 最优解释路径
- 必要的状态推断结果

笨鸟 v0.1 的目标不是聊天，不是续写，也不是大语言模型替代品。
它的任务是把语言描述还原成结构化场景认知结果。

## 2. 明确做什么、不做什么

### 做什么

笨鸟 v0.1 只做以下四件事：

1. 语义拆分
   将输入文本拆成多个初始子信号，包括对象信号、属性信号、关系信号、事件信号、场景信号。
2. 稀疏激活
   根据子信号只激活相关空间，而不是全局穷举处理。
3. 多假设竞争
   允许多个候选场景解释同时存在，并通过全局评分选出最优解释。
4. 结构化输出
   输出对象—属性—关系—事件解释图，并附解释路径。

### 不做什么

笨鸟 v0.1 明确不做以下任务：

- 开放域问答
- 长文本理解
- 自由对话
- 常识百科全覆盖
- 图像输入
- 连续物理仿真
- 复杂多轮规划
- 端到端生成
- 无限结构生长

这些边界在 v0.1 中固定，不再漂移。

## 3. 固定使用场景

### 场景名称

受限场景描述语义理解。

### 核心场景定义

用户输入一段描述局部现实场景的短文本，笨鸟识别其中的对象、属性、关系、事件，并判断最可能的场景类型或场景解释。

### 典型应用类型

v0.1 只覆盖以下类型的句子：

- 描述某处堆放情况
- 描述对象之间的位置关系
- 描述局部异常状态
- 描述某个简单事件导致的场景变化
- 描述疑似风险、杂乱、污染、施工、堆积等场景

### 典型输入示例

- “屋后空地上堆着碎砖和木板，旁边停着一辆小卡车，地面裸露，没有整齐堆放。”
- “桌子左边有一个红色杯子，杯子里有水，猫碰到了杯子。”
- “路边沟渠里有一片发黑的积水，周围有杂草，靠近居民房。”
- “院子角落堆着塑料袋和生活杂物，旁边有翻倒的垃圾桶。”

### 典型输出示例

输出目标不是一句自然语言，而是结构化结果，例如：

```json
{
  "objects": ["空地", "碎砖", "木板", "小卡车"],
  "attributes": [
    {"target": "碎砖", "value": "破碎"},
    {"target": "碎砖", "value": "堆积"},
    {"target": "木板", "value": "散乱"},
    {"target": "地面", "value": "裸露"},
    {"target": "场景", "value": "不整齐"}
  ],
  "relations": [
    {"source": "碎砖", "type": "在……上", "target": "空地"},
    {"source": "木板", "type": "在……上", "target": "空地"},
    {"source": "小卡车", "type": "邻近", "target": "堆放区"}
  ],
  "events": [],
  "scene_hypotheses": [
    {"label": "建筑废弃物堆放", "score": 0.83},
    {"label": "临时施工材料堆放", "score": 0.64},
    {"label": "普通杂物堆积", "score": 0.29}
  ],
  "best_hypothesis": {"label": "建筑废弃物堆放", "score": 0.83},
  "reasoning_path": [
    "碎砖 -> 建筑材料相关",
    "裸露地面 + 不整齐堆放 + 邻近小卡车 -> 施工/废弃场景增强",
    "全局竞争后建筑废弃物堆放胜出"
  ]
}
```

## 4. 固定领域边界

v0.1 只支持一个受限语义领域。

### 领域名称

局部场景异常与堆放理解领域。

### 固定语义中心

#### 对象类

- 容器类：杯子、桶、箱子、袋子
- 堆放物类：砖块、木板、塑料袋、垃圾、杂物
- 环境类：地面、桌子、道路、沟渠、院子、空地
- 附属物类：车辆、房屋、树木、垃圾桶

#### 属性类

- 规则 / 不规则
- 整齐 / 散乱
- 完整 / 破碎
- 干净 / 污染
- 发黑 / 灰白 / 混杂
- 稳定 / 倾斜 / 翻倒
- 裸露 / 覆盖

#### 关系类

- 在……上
- 在……旁边
- 在……里面
- 靠近
- 邻接
- 覆盖
- 分离
- 堆放于
- 散落于

#### 事件类

- 堆放
- 散落
- 倾倒
- 翻倒
- 撞击
- 泄漏
- 覆盖
- 暴露

#### 场景解释类

- 建筑废弃物堆放
- 生活杂物堆放
- 污染 / 异常积水
- 普通物体摆放
- 临时施工场景
- 倾倒 / 翻倒事件场景

## 5. 输入规范

### 输入形式

只接受以下两种：

- 形式 A：单句场景描述
- 形式 B：两到三句短场景描述

### 输入长度限制

固定限制为：

- 最短：5 个词或 1 个短句
- 最长：3 句，或约 60 个中文字符的有效描述

v0.1 不处理长文本。

### 输入内容限制

输入必须满足：

- 是场景描述，不是议论文
- 不是抽象哲学句子
- 不是纯情绪表达
- 不要求多轮对话上下文
- 不含大量隐喻、反讽、文学修辞

## 6. 输出规范

### 输出主格式

固定输出为六个部分：

1. 对象列表
2. 属性列表
3. 关系列表
4. 事件列表
5. 候选场景解释
6. 解释路径摘要

### 输出风格

v0.1 默认输出为结构化 JSON 风格或等价内部结构，不以自由文本为主。

建议统一成以下结构：

```json
{
  "objects": [],
  "attributes": [],
  "relations": [],
  "events": [],
  "scene_hypotheses": [],
  "best_hypothesis": {},
  "reasoning_path": []
}
```

## 7. 固定内部设计

### 初始子信号类型固定为五类

1. 对象信号
   例如：碎砖、木板、杯子、猫、水、沟渠。
2. 属性信号
   例如：红色、发黑、破碎、散乱、裸露、翻倒。
3. 关系信号
   例如：在……上、邻近、在……里、靠近。
4. 事件信号
   例如：撞到、堆放、泄漏、翻倒。
5. 场景提示信号
   例如：院子角落、路边、沟渠里、屋后空地。

### 空间类型固定为四类

- 概念空间
  表示“是什么”，例如垃圾、建筑废弃物、杂物、杯子、水体、车辆。
- 属性空间
  表示“有什么特征”，例如破碎、散乱、稳定、翻倒、黑色、裸露。
- 关系空间
  表示“和谁怎样关联”，例如在上方、在旁边、在内部、覆盖、邻接。
- 场景空间
  表示“属于什么局部场景解释”，例如废弃堆放场景、普通摆放场景、倾倒事故场景、污染积水场景、临时施工场景。

注意：
v0.1 暂不单独拆出事件空间。
事件先由关系与状态变化机制承载，最终归并到场景空间解释中。

### 推理流程固定为六步

1. 文本解析
   从输入文本抽取初始对象、属性、关系、事件线索。
2. 生成初始子信号
   形成五类信号包。
3. 空间激活
   子信号只激活匹配的相关空间。
4. 局部传播
   被激活空间向相关空间传播支持或抑制信号。
5. 候选场景竞争
   形成多个场景解释候选，并进行全局打分。
6. 输出结构化解释
   输出最优结果及理由链。

## 8. 评分机制

### 局部激活评分

用于判断一个信号是否足以激活一个空间，由三部分组成：

- 类型匹配
- 内容相似
- 上下文兼容

### 全局假设评分

一个场景候选的总分由以下正向项组成：

- 对象支持度
- 属性一致性
- 关系一致性
- 场景兼容性
- 事件合理性

并减去以下惩罚项：

- 冲突惩罚
- 互斥惩罚
- 路径复杂度惩罚

### 输出候选数量

固定规则：

- 保留 top-3 场景解释
- 输出其中 top-1 作为最终结果

## 9. 复杂度约束

为避免传播爆炸，v0.1 固定以下上限：

- 每个初始信号最多激活 5 个空间
- 每个空间最多向 3 个关联空间继续传播
- 最大传播深度 4 轮
- 同时保留的候选场景解释不超过 10 个
- 最终输出候选解释为 3 个

这些数字在 v0.1 中先固定，不做开放调参。

## 10. 学习机制

v0.1 不是完全静态系统，但学习必须受控。

### 允许学习的内容

- 新词到已有概念空间的映射
- 属性空间阈值微调
- 关系边权重微调
- 场景解释权重微调
- 临时概念挂接到已有概念树

### 不允许学习的内容

- 无限新建空间
- 自由改写全部结构
- 完全无监督生长
- 自动扩张到新领域

### 空间生长规则

v0.1 只允许三种空间状态：

- 稳定空间
- 候选空间
- 临时空间

只有当一个新概念多次出现、且无法被现有空间稳定解释时，才允许从临时空间升级为候选空间，再升级为稳定空间。

## 11. 首批测试任务

### 任务组 A：静态堆放场景理解

示例：

- “空地上堆着碎砖和木板，旁边停着小卡车。”
- “院角堆着塑料袋和旧纸箱，摆放杂乱。”

目标：

判断是建筑废弃物、生活杂物、普通堆放，还是临时施工材料。

### 任务组 B：简单位置关系理解

示例：

- “红色杯子放在桌子左边，杯子里有水。”
- “垃圾桶在门口旁边，袋子堆在垃圾桶后面。”

目标：

抽取对象、属性和位置关系。

### 任务组 C：简单事件导致的状态变化

示例：

- “猫碰到了杯子，杯子翻倒了，水洒在桌面上。”
- “垃圾桶被碰倒，垃圾散落在地上。”

目标：

- 抽取事件链
- 更新对象状态
- 输出对应场景解释

## 12. 成功标准

### 最低成功标准

满足以下条件即可认为 v0.1 成功：

- 能把短场景句拆成对象、属性、关系、事件信号
- 能输出 top-3 候选场景解释
- 能给出结构化解释路径
- 能对简单冲突解释做出区分
- 能在小范围新词输入下完成受控映射或临时挂接

### 非成功标准

v0.1 不以下列目标作为成功标准：

- 超越大语言模型
- 通用推理
- 自主发现世界规律
- 高级创造性
- 大规模知识迁移

## 13. 后续设计的固定前提

从现在开始，后续所有笨鸟 v0.1 设计都默认遵守以下前提：

- 输入是受限短场景描述
- 核心任务是对象—属性—关系—事件解释
- 输出是结构化场景解释
- 领域是局部场景异常与堆放理解
- 传播是稀疏、有限深度、有限候选
- 学习是受控、有限生长

这就是 v0.1 的边界，不再漂移。

## 14. 版本结论

这版方案的优点是：

- 足够小，能够落地
- 足够接近笨鸟的原始思想
- 能体现“多子信号—空间激活—传播—竞争”的核心机制
- 后续可以平滑升级到结构化场景理解、图像前端接入，以及语义加离散物理的组合能力

因此，这一版可以作为笨鸟真正的第一版规范。
