# 笨鸟神经网络形式化规范

本文档描述的是笨鸟神经网络的一般架构层规范。
如果涉及当前版本的具体边界、任务和成功标准，应以 [笨鸟v0.1规范](d:/SB/文档/笨鸟v0.1规范.md) 为准。

本文档将笨鸟神经网络（SB）的概念整理成更明确的工程化规范。
它仍然属于研究型架构，但目标是减少执行过程、学习机制和生命周期管理上的歧义。

## 1. 系统不变式

笨鸟神经网络应尽量保持以下不变式：

1. 推理必须是稀疏的，而不是全局密集展开
2. 记忆空间应当通过内容被激活，而不仅仅通过固定编号访问
3. 重复语义信号必须被抑制，除非它带来了新的有效证据
4. 每一步传播都必须消耗可计量的预算
5. 空间创建应当由新颖度触发，而不是由每次观察无条件触发

## 2. 运行时对象

### 信号

信号是事件式的信息载体。
它在神经元空间和工作记忆之间流动。

必要字段包括：

- 信号编号
- 信号类型
- 信号取值
- 标签集合
- 向量表示
- 能量
- 置信度
- 生存步数
- 来源路径

字段含义如下：

- 能量：剩余传播预算
- 置信度：该信号当前的语义可信度
- 生存步数：剩余跳数
- 来源路径：沿空间图传播时形成的来源路径

### 空间

空间是持久化的“计算-记忆”单元。
每个空间至少包含以下内容：

- 敏感特征签名
- 局部变换规则或局部学习模块
- 指向后继空间的转移统计
- 使用统计与奖励统计

空间类型可以包括：

- 概念空间
- 关系空间
- 操作空间
- 情景空间

### 假设

假设是输入的一种局部解释。
它通常包含：

- 当前已接受的信号集合
- 累积分数
- 传播轨迹
- 最近激活过的空间序列

## 3. 执行阶段

笨鸟神经网络的推理过程可以分成六个阶段。

### 阶段 A：输入分解

外部输入首先被分解成少量主信号。
这些主信号可能对应：

- 实体
- 动作
- 属性
- 角色
- 上下文线索

### 阶段 B：稀疏路由

对每个活跃信号，执行以下步骤：

1. 通过标签索引或向量索引收集候选空间
2. 计算基础匹配分数
3. 注入来自近期成功空间的转移加成
4. 仅保留有限个最佳候选

推荐的路由分数形式：

```text
路由分数 =
  基础匹配 +
  转移加成 -
  局部代价
```

### 阶段 C：局部发射

每个活跃空间都可以发射零个或多个子信号。
只有当局部规则或局部模块条件满足时，发射才会发生。

### 阶段 D：相关性门控

每个子信号都要与当前工作记忆进行匹配。
只有高于相关性阈值的信号才允许保留。

### 阶段 E：签名去重

如果子信号的语义签名已经存在于当前假设中，则不再重复加入。
这一步用于阻断置信度在局部回路中的无意义放大。

### 阶段 F：束搜索筛选

所有候选假设都会被打分和剪枝。
只有当前最强的一批局部解释会进入下一轮。

## 4. 预算模型

笨鸟神经网络必须运行在显式预算控制下。

推荐控制的预算维度包括：

- 推理步数
- 束宽
- 单个信号的候选空间上限
- 单个空间的最大发射数
- 单个假设保留的最大信号数
- 最大扩展次数

预算耗尽不是错误，而是正常的停止条件。

## 5. 空间生命周期

### 创建

当满足以下条件时创建新空间：

- 没有足够强的已有空间可以匹配
- 当前片段具有保留价值
- 新颖度估计持续偏高

### 更新

当满足以下条件时更新已有空间：

- 该空间与当前片段匹配良好
- 当前片段能够改进它的特征签名或局部变换能力

### 合并

当满足以下条件时合并空间：

- 敏感特征签名高度重叠
- 输出模式高度重叠
- 转移邻域高度重叠

### 拆分

当满足以下条件时拆分空间：

- 单个空间长期承载互不兼容的模式
- 更新之后误差仍然偏高
- 发射模式能够稳定聚成多个簇

### 归档 / 剪枝

当满足以下条件时归档或剪枝空间：

- 使用率长期偏低
- 奖励表现持续较弱
- 附近存在明显更强的替代空间

## 6. 学习循环

最小在线学习循环可以写成：

1. 对当前片段执行推理
2. 记录成功路径和奖励
3. 对成功路径中的空间转移进行强化
4. 更新匹配到的空间
5. 如果新颖度仍高，则创建新空间
6. 周期性执行合并、拆分和剪枝

## 7. 转移先验

转移先验是“存储参与计算”这一思想中的核心部分。
每个空间都会记录自己倾向于激活哪些后继空间。

在推理时：

- 近期成功空间会影响下一跳的路由偏好
- 转移先验可以提高分数，但不能绕过阈值
- 当某种转移长期失效时，这种先验应当衰减

因此，已经写入存储的空间图结构，会在实时推理中直接改变计算路径。

## 8. 建议研究问题

当前原型已经足够支撑一些早期研究问题：

1. 随着空间数量增长，路由质量会在何时明显改善
2. 转移先验在何时提供帮助，何时开始过拟合
3. 哪种合并与拆分准则更有利于系统稳定
4. 签名去重与预算控制究竟节省了多少计算量
5. 局部可学习模块是否优于手工规则变换

## 9. 评估指标

评估笨鸟神经网络时，可以重点关注：

- 任务准确率
- 单次查询激活的空间数量
- 单次推理的平均扩展次数
- 平均传播深度
- 记忆空间增长速率
- 合并发生率
- 剪枝发生率
- 回路抑制率

## 10. 最小原型映射

当前仓库中的对应关系如下：

- `sb/core.py`：运行时对象、稀疏推理和空间管理接口
- `examples/minimal_demo.py`：多信号传播的玩具示例
- `笨鸟神经网络说明.md`：整体架构说明

当前原型还不是最终版本的笨鸟神经网络。
它更像是一个可运行的脚手架，用来验证这些设计假设是否站得住。
